## Algorithm Summary

### Round Robin (Static Baseline)
- **Strategy**: Cyclic assignment, no feedback
- **Strengths**: Lowest overhead, predictable
- **Weakness**: Ignores node latency and load

### Least Connections (Heuristic)
- **Strategy**: Route to node with fewest active connections
- **Strengths**: Adapts to connection-level load
- **Weakness**: No latency awareness

### UCB (Multi-Armed Bandit)
- **Strategy**: Balances exploitation (fast nodes) with exploration (underused nodes)
- **Reward**: `1 / (1 + latency_ms / 1000)` — lower latency = higher reward
- **Strengths**: Learns node performance over time; adapts to changing conditions
- **Weakness**: Initial exploration overhead; slower convergence with many nodes

### Metric-Aware (Adaptive)
- **Strategy**: Weighted scoring using push-based metrics (latency, queue depth, connections, failure rate)
- **Strengths**: Richest signal set; incorporates multiple operational dimensions
- **Weakness**: Higher computational overhead; sensitive to weight tuning and metric staleness

## Methodology Notes

- All experiments run on containerized infrastructure (Docker Compose)
- 6 heterogeneous nodes with varying delay profiles (50ms–1500ms)
- Nodes 2 and 5 have crash simulation (3–5%); Node 3 has timeout simulation (5%)
- Controller timeout: 1 second per node attempt; retries all nodes before failing
- Each (algorithm, workload, pattern) combination repeated 3 times
- Clean Docker restart between algorithm changes
- All results generated by `scripts/benchmark_matrix.py`
